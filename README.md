# 藏品展仓模拟器

**Collection Showcase Simulator**

这是一个面向游戏藏品生成与展示流程的本地模拟项目。它以配表作为权威数据源，根据资源预算、品质权重和道具属性生成一局藏品结果，再将结果排入动态网格，用于观察藏品规模、价值分布、空间形态以及逐件展示时的视觉节奏。

项目同时提供生成工作区和配置工作区。生成工作区用于反复试验参数与查看结果；配置工作区用于维护品质、规格、道具、价值资源区间和预算档位。生成结果不写入数据库，适合快速调整规则、复现指定随机结果和验证展示方案。

整个项目使用 Python 3.8 标准库与原生 HTML、CSS、JavaScript 实现，不依赖第三方运行包、数据库或前端构建工具。配置保存在 `data/` 下的 JSONL 与 JSON 文件中，浏览器通过本地 HTTP 接口读取和保存数据。

## 运行

```powershell
python app.py
```

访问 <http://127.0.0.1:8000>。

## 测试

```powershell
python -m unittest discover -s tests -v
```

测试主要覆盖配置校验、种子复现、资源预算生成、方向与排布合法性、接口行为以及前端关键交互约束。

## 数据来源

示例道具数据由 `tasks/import_orzice_collection.py` 从 [Orzice 收集品页面](https://orzice.com/v/collection) 导入。更新时，导入器会按道具 ID 合并并保留已有规格与本地新增记录，再以原子替换方式写入目标文件；项目运行本身不依赖该站点。
